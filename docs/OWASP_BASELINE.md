# OWASP Baseline Checklist

O BirthHub 360 compromete-se a seguir as melhores práticas de segurança de software, adotando o [OWASP Top 10 (2021)](https://owasp.org/Top10/) como linha de base para auditorias contínuas. Esta checklist mapeia os riscos com as políticas de código de nossos frameworks principais (Express, FastAPI e Prisma) e registra o status atual da proteção.

## Status Inicial do Baseline (Ciclo 1)

| Categoria OWASP                                         | Vetor / Risco no Contexto do BirthHub                                                                                | Status Atual                    | Ferramenta / Mitigação Implementada                                                                                                                                                                                                       |
| ------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- | ------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **A01:2021-Broken Access Control**                      | Um usuário de um Tenant visualizar leads, faturas ou relatórios de outro Tenant. Acesso a endpoints de Admin.        | 🟢 Controlado                   | O `api-gateway` e o ORM Prisma garantem a passagem do `tenant_id` em todos os contextos autenticados (Row Level Security conceitual). Middleware de Autorização restrito por Roles.                                                       |
| **A02:2021-Cryptographic Failures**                     | Exposição de chaves de API, senhas ou processamento não seguro de pagamentos.                                        | 🟢 Controlado                   | Adoção do `ADR-004` (Secrets Vaults). TLS obrigatório em todas as rotas de trânsito de rede. Redação de PII ativa nos Logs (`ADR-003`).                                                                                                   |
| **A03:2021-Injection**                                  | ORM SQL Injections, e Prompt Injections nos 8 Agentes LLMs (ex: LLM gerando scripts maliciosos).                     | 🟡 Atenção (Acompanhamento)     | O ORM Prisma impede SQL injection base (Prepared Statements). _Porém_, as chamadas de LangGraph e Prompts necessitam de validação constante de Output via `Pydantic` e remoção de tags HTML/Javascript do LLM para evitar XSS secundário. |
| **A04:2021-Insecure Design**                            | Modelos lógicos que permitem by-pass de negócio (ex: aprovar negócio sem limite de crédito).                         | 🟢 Controlado                   | Arquitetura Multi-Agente requer aprovação multi-etapas.                                                                                                                                                                                   |
| **A05:2021-Security Misconfiguration**                  | Headers de segurança ausentes, diretórios expostos, senhas "default" ativas no DB do Cloud Run.                      | 🟢 Controlado                   | Uso do Helmet.js no `api-gateway`, dependência severa da CI/CD, não há acessos "default" providos no `terraform`.                                                                                                                         |
| **A06:2021-Vulnerable and Outdated Components**         | Pacotes NPM (`express`, etc) ou pacotes PyPI (`langchain`, `pydantic`) defasados possuindo CVEs ativas.              | 🟡 Precisa Melhoria             | Dependabot ativo no repositório. O processo de "update" precisa ser estabelecido formalmente.                                                                                                                                             |
| **A07:2021-Identification and Authentication Failures** | Força Bruta em login, Sessões que não expiram.                                                                       | 🟢 Controlado                   | Utilização de provedor de Auth consolidado (Supabase/Auth0) com proteção de brute force e MFA embutidos. JWT com vida útil curta configurada (15m).                                                                                       |
| **A08:2021-Software and Data Integrity Failures**       | Modificações não assinadas no CI/CD, execução de dependências OpenSource maliciosas.                                 | 🟢 Controlado                   | Pipeline fechado (Branch Protection + CODEOWNERS + Dependências via `pnpm-lock.yaml` estritas).                                                                                                                                           |
| **A09:2021-Security Logging and Monitoring Failures**   | Ausência de trilha de auditoria em ações críticas (Ex: Agente apagando Lead).                                        | 🟢 Controlado                   | Uso estrito de Log Estruturado com `trace_id` e a tabela `AgentLog` registrando o Audit Trail de cada passo do LangGraph.                                                                                                                 |
| **A10:2021-Server-Side Request Forgery (SSRF)**         | Ferramentas (Tools) do Agente acessando IPs internos ou a rede local do Kubernetes explorando APIs não-documentadas. | 🟡 Atenção (Risco Arquitetural) | Os agentes precisam fazer requests web externos (Enrichment). Os _Tools_ em Python devem usar clientes HTTP blindados contra a resolução de endereços internos (ex: bloqueio de acessos ao range `169.254.169.254` e `10.0.0.0/8`).       |

## Regras

- Nenhuma feature será aprovada se o revisor identificar violação óbvia aos Top 10 (especialmente A01, A03 e A10 que são frequentes em Agentic AI).
